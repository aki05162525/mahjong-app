# v1.2.0 - 認証・認可の導入

## 麻雀大会スコア v1.2.0

管理者パスワード方式を廃止し、Google OAuth による認証・認可に移行したリリース。大会ごとにオーナーが決まり、オーナーのみが管理操作を行えるようになった。

### 主な変更

#### 認証・認可（Supabase Auth + Google OAuth）

- Google アカウントでログイン・ログアウトできるようになった
- 大会作成にはログインが必要になった（未ログイン時は 401）
- 選手・卓の登録削除、対局削除、大会削除はオーナー（作成者）のみ可能（他ユーザーは 403）
- 対局入力は従来どおり認証不要で誰でも可能
- `tournaments` テーブルに `owner_id` カラムを追加し RLS で制御
- `ADMIN_PASSWORD` によるアクセス制御をコードから完全削除

#### UI の変更

- トップページ：未ログイン時は「Google でログイン」ボタン、ログイン後は自分の大会一覧と作成フォームを表示
- 大会ページ：オーナー本人のみ管理 UI（選手・卓登録フォーム、削除ボタン）を表示

#### バグ修正

- ユーザー切り替え時に前ユーザーの大会が一瞬表示される競合状態を解消
- `.single()` のエラーを握りつぶしていたバグを修正

#### テスト

- 認証境界テストを追加（未ログインで 401、別ユーザーで 403 を返すことを検証）

### 設計の方針

大会オーナー制にした理由は、パスワードを知っている人なら誰でも全大会を削除できるという旧方式の穴を塞ぐため。対局入力は身内の複数人が行うユースケースを維持したまま、管理操作だけをオーナーに絞る設計にした（`docs/adr/0001-google-oauth-only.md`・`0002-match-input-no-auth.md` 参照）。
