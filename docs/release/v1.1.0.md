# v1.1.0 - Supabase 移行

## 麻雀大会スコア v1.1.0

データベースを Firebase（Firestore）から Supabase（PostgreSQL）に全面移行したリリース。ユーザー向け機能の見た目は変わらないが、基盤が刷新されている。

### 主な変更

#### バックエンド移行（Firebase → Supabase）

- Firestore を完全廃止し、PostgreSQL（Supabase）に移行
- スキーマ・RLS ポリシーを新規設計（クロス大会汚染防止・重複結果防止）
- 全書き込みを API Routes 経由に統一し、`service_role` キーをサーバーサイドのみで管理
- サーバーサイドバリデーション追加（スコア合計 100,000 点チェックなど）
- 移行スクリプトで既存データを Firestore → Supabase へ移送

#### 開発環境

- パッケージマネージャーを npm → pnpm に移行
- GitHub Actions CI を導入（lint・test・型チェック）
- lefthook を導入し、コミット前に Prettier・ESLint を自動実行
- `src/lib/` を責務ごとに分割し、Supabase クライアントを `src/infra/` へ移動

#### バグ修正

- リネーム後に対局履歴が古い表示名を保持し続ける問題を修正
- `matches → tables` JOIN の外部キー曖昧性エラーを解消

#### テスト

- rate-limit・players・matches の API バリデーションテストを追加

### 移行の方針

セキュリティ・スケーラビリティの観点から Supabase を採用。RLS でテーブルレベルの読み取り制御を行いつつ、書き込みは全て API Routes に集約してサービスキーを秘匿する構成にした。
