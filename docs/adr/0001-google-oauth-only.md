# Google OAuth のみで認証する

認証方式として Supabase Auth + Google OAuth を採用し、メール/パスワードやマジックリンクは提供しない。

このアプリの利用者は少人数の麻雀仲間であり、全員が Google アカウントを持っていることを前提にできる。メール/パスワード方式は「パスワード忘れ」対応が必要になり運用コストが上がるだけで、Google OAuth に対してメリットがない。

## Considered Options

- メール＋パスワード: パスワードリセットフローが必要になる
- マジックリンク: メールが届かない環境では詰まる
- Google OAuth: パスワード管理不要、スマホでも使いやすい ← 採用
